# 🏦 FinTetsu（フィン鉄）ゲーム設計書 v0.1

> **コンセプト：** 日本全国の都市を巡りながら銀行支店を買収・経営し、地域経済を育てて資産日本一を目指すボードゲーム型ブラウザゲーム

---

## 1. ゲーム概要

| 項目 | 内容 |
|------|------|
| プレイ人数 | 2〜4人（対戦 or CPU） |
| プレイ時間 | 約30〜60分（ショートモード）/ 最大100ターン |
| 対象 | 金融・経済に興味があるビジネスパーソン・学生 |
| 目標 | ゲーム終了時に総資産（現金＋支店評価額＋地域株）が最大のプレイヤーが勝利 |

---

## 2. 世界観・テーマ

- **舞台：** 日本の実在都市をモデルにしたマップ（北海道〜沖縄）
- **プレイヤー：** 新興金融グループの頭取として日本征服を狙う
- **対抗勢力：** 既存のメガバンク（NPC）が各都市を支配しており、買収や競合が起きる
- **地域色：** 各都市は「産業タイプ」を持ち、経済イベントと連動して繁栄・衰退する

---

## 3. マップ設計

### 3-1. マップ構造

```
[札幌] --- [仙台] --- [新潟] --- [東京]
                                   |
[福岡] --- [大阪] --- [名古屋] ---[横浜]
  |
[那覇]
```

- マスはノード（都市）とエッジ（路線）で構成
- サイコロを振って移動（1〜6）
- ルートは複数あり、プレイヤーが選択できる

### 3-2. 都市タイプ（産業分類）

| タイプ | 代表都市 | 特徴 |
|--------|----------|------|
| 工業都市 | 名古屋・北九州 | 製造業ローン需要が高い |
| 観光都市 | 京都・那覇・函館 | 季節イベントで収益変動 |
| 農業都市 | 帯広・熊本 | 農業融資・地域創生イベントが多い |
| 金融都市 | 東京・大阪 | 高コストだが収益最大・メガバンクの本拠地 |
| 過疎地域 | 鳥取・高知・島根 | 安価だが特殊ボーナスあり（地方創生補助金） |

---

## 4. プレイヤーの行動

### 4-1. ターンの流れ

```
① サイコロを振る（1〜6）
② 移動先を選ぶ（分岐あり）
③ 都市マスに着く
    ├─ 未所有マス → 支店を設立できる
    ├─ 自分の支店 → 支店強化・サービス拡充
    ├─ 他者の支店 → 手数料を支払う（利用料）
    └─ イベントマス → カードを引く
④ カード効果・収入計算
⑤ 次のプレイヤーへ
```

### 4-2. 支店の設立・強化

各都市に設立できる支店にはレベルがある。

| レベル | 名称 | 初期費用 | 利用料（他者） | 収益/ターン |
|--------|------|----------|----------------|-------------|
| Lv1 | ATM出張所 | 50万 | 5万 | 2万 |
| Lv2 | 支店 | 200万 | 20万 | 8万 |
| Lv3 | 主要支店 | 500万 | 60万 | 25万 |
| Lv4 | 地域本部 | 1,200万 | 150万 | 80万 |
| Lv5 | 統括本部 | 3,000万 | 400万 | 250万 |

> 同一都市に自分の支店が複数あると「地域独占ボーナス」（収益+30%）が入る

## 5. 銀行・金融システム

### 5-1. 融資（ローン）システム

- 資金が足りないとき、銀行から融資を受けられる
- 借入金には**金利**がかかり、毎ターン利息が発生
- 金利は「景気イベント」「日銀政策金利カード」で変動

```
借入可能額 = プレイヤーの総資産 × 0.5（担保率）
デフォルト判定 = 現金残高がマイナスになると強制売却
```

### 5-2. 地銀 vs メガバンクの違い

| 属性 | 地方銀行 | メガバンク |
|------|----------|------------|
| 拠点コスト | 低い | 高い |
| 収益力 | 中程度 | 高い |
| 地域イベントボーナス | **大きい** | 小さい |
| 合併・提携 | しやすい | しにくい |
| 特殊能力 | 「地方創生カード」使用可 | 「フィンテック提携カード」使用可 |

> プレイヤーは序盤「地銀戦略」か「メガバンク戦略」かを選ぶことになる

### 5-3. 銀行タイプ選択（ゲーム開始時）

| タイプ | 初期資金 | 特徴 |
|--------|----------|------|
| 地方銀行 | 300万 | 地域イベントで+50%ボーナス、農業・過疎地が安く買える |
| メガバンク | 500万 | 都市部で有利、手数料が割高に設定できる |
| ネット銀行 | 250万 | 移動コストゼロ、どの都市にも即日設立可能 |
| 信用金庫 | 200万 | 合併コストが激安、他プレイヤーと提携でき収益シェア |

## 6. イベントシステム

### 6-1. カード種別

#### 🎴 チャンスカード（良いイベント）
- 「日銀が金利を引き下げ → 融資コスト半額（3ターン）」
- 「地方創生補助金 → 過疎地支店に500万補助」
- 「フィンテック提携 → 選択した都市の収益2倍（1ターン）」
- 「M&A成功 → 他プレイヤーの支店を市場価格の80%で強制買収」

#### 💀 ハプニングカード（悪いイベント）
- 「不正融資発覚 → 一番高い支店を一時停止（3ターン）」
- 「取り付け騒ぎ → 全収益が50%ダウン（2ターン）」
- 「地震・災害 → 指定都市の支店価値が30%ダウン」
- 「マイナス金利導入 → 全プレイヤーの収益-10%（5ターン）」

#### 🏙️ 地域経済カード（地域連動）
- 「農業不作 → 農業都市の支店収益半減」
- 「インバウンド回復 → 観光都市の収益2倍」
- 「工場移転 → 工業都市1つがゴーストタウン化（収益ゼロ）」
- 「移住促進キャンペーン → 過疎地域の人口増加 → 収益UP」

### 6-2. 景気インジケーター

ゲーム全体に「景気ゲージ」があり、イベントで変動する。

```
[不況] ←――――――――――→ [好況]
  1    2    3    4    5
       ↑現在地
```

| 景気レベル | 融資金利 | 支店収益 | 支店売買価格 |
|------------|----------|----------|--------------|
| 不況（1-2） | 高い | -20% | 安い（買い時） |
| 普通（3） | 標準 | 標準 | 標準 |
| 好況（4-5） | 低い | +20% | 高い（売り時） |

## 7. 地域経済システム（地方創生要素）

### 7-1. 地域人口・産業指数

各都市は「人口」「産業活力」の2つのパラメータを持つ。

```
人口指数：10〜100（高いほど支店収益UP）
産業活力：10〜100（高いほど融資需要UP → 手数料収入UP）
```

### 7-2. 地域育成アクション

支店を持つプレイヤーは都市を育てられる（任意行動）：

| アクション | コスト | 効果 |
|------------|--------|------|
| 産業融資 | 100万 | 産業活力+10 → 収益UP |
| 移住支援 | 80万 | 人口+5 → 長期的に収益UP |
| 観光開発 | 150万 | 観光都市化 → 季節イベントボーナス解放 |
| スタートアップ支援 | 200万 | フィンテック企業が育つと支店収益×1.5 |

> **地方創生スコア**が高いプレイヤーは「地方創生大賞」ボーナス（ゲーム終了時+500万）

### 7-3. 関係人口システム

移住しなくても都市と関わる「関係人口」を増やせる。
- 関係人口が増えると「地域応援ローン需要」が発生
- 過疎地でも関係人口が多ければ支店収益が維持される

## 8. 合併・提携システム

### 8-1. 銀行合併

- 2プレイヤーが合意すると「合併」できる（協力プレイ要素）
- 合併後は両プレイヤーの資産を合算して戦略を立てられる
- 「コングロマリット形成」で特殊カードが使えるようになる

### 8-2. 業務提携

合併より軽い連携：
- フィンテック提携 → 指定路線の移動コスト半額
- 地銀ネットワーク → 地方都市の手数料収入をシェア
- 証券提携 → 毎ターン「地域株」配当を受け取れる

---

## 9. 勝利条件・終了条件

### 9-1. 通常勝利

- **100ターン経過**でゲーム終了 → 総資産最大プレイヤーの勝利
- 総資産 = 現金 + 支店評価額 + 保有株 - 借入残高

### 9-2. 特殊勝利（即時終了）

| 条件 | 勝利者 |
|------|--------|
| 総資産1億円達成 | 最初に達成したプレイヤー |
| 全都市の50%以上を支配 | そのプレイヤー（独占勝利） |
| 他全プレイヤーが破産 | 生き残ったプレイヤー |

### 9-3. 破産条件

- 現金残高がマイナスになり、売れる資産もない状態
- 破産するとゲームから脱落、支店はオークションにかかる

## 13. 中核の作り直し：システミックリスク（v0.2 方向転換）

> **課題：** v0.1 の中核ループ（サイコロ→移動→物件購入→通行料→不労所得）は
> 桃鉄/モノポリーの骨格そのもので、「銀行であること」がプレイ中の判断に
> 効いていなかった。景気ゲージも収益±20%のランダム漂流にすぎず意思決定がない。
>
> **方針：** 中核の緊張を、銀行にしか作れない **「繋がっているから一緒に沈む」=
> システミックリスク（連鎖倒産）** に置き換える。盤・サイコロ・支店は
> 「資本を展開して利回りを生む層」として残す。

### 13-1. インターバンク市場（連鎖グラフ）

- 銀行（プレイヤー）は銀行間市場で貸し借りでき、各ローンは
  「債権者 → 債務者」の有向エッジ。**この網がそのまま倒産連鎖の伝播経路**になる。
- 余剰現金を貸せば金利（利回り）を得るが、その相手の破綻リスクを背負う
  （利回り vs カウンターパーティ・リスク）。

### 13-2. 自己資本比率と信用格付け

- `純資産 = 現金 + 支店評価額 + 貸付債権 − 借入負債`
- `自己資本比率 = 純資産 / 総資産`。規制ライン 8%（BIS 規制のもじり）を割ると危険水域。
- 比率から信用格付け（AAA〜D）を算出。格付けは借入コストと信用を左右する。

### 13-3. デフォルト連鎖（取り付け・債務超過）

- **流動性危機：** 現金がマイナスになり、銀行間市場からの自動借入でも埋まらない → 破綻。
- **債務超過：** 純資産がマイナス → 破綻。
- 破綻すると支店は接収され、絡む全ローンが消滅する。**破綻行へ貸していた債権者は
  債権（資産）を失い、それで純資産割れすれば次の行も破綻する＝連鎖**。
- 他行がすべて破綻すれば、最後の1行が即勝利（連鎖倒産による勝利）。

### 13-4. 実装済みの最小スライス（Step 13）

`store/gameStore.ts` に以下を実装：

- インターバンク融資 `loans`（債権者→債務者エッジ）
- 利用料・金利を現金で払えないときの **自動借入**（`coverShortfall`）
- 1 ラウンド 6% の **インターバンク金利**（借り換えで雪だるま式に膨張）
- **デフォルト連鎖**の不動点解決（`resolveDefaults`）と破綻者のターン除外
- 自己資本比率・格付けセレクタ（`capital` / `debtOf`）と HUD 表示

> **次の拡張候補：** 手動の与信判断（誰にいくら貸すか）UI、揺らいだ相手から
> 与信を引く＝取り付け誘発、危機時の投げ売り資産の底値買い、中央銀行（最後の貸し手）、
> 危機（ストレステスト）をクライマックスに据えた勝利条件。

## 14. 世界観の再構成：「中小企業の社長」（v0.3）

> **方針：** プレイヤーを「銀行の頭取」から **「中小企業の社長」** に変更。
> 連鎖倒産は本来、銀行間融資より **取引先の倒産で売掛金が回収不能になる**
> 中小企業の現象として典型的で、社長コンセプトに最も自然に載る。
> v0.2 で作った連鎖倒産機構をそのまま、より腑に落ちる形に読み替える。

### 14-1. 用語の読み替え

| v0.1〜v0.2（銀行） | v0.3（中小企業の社長） |
|---|---|
| 頭取 / 銀行 | 社長 / 会社 |
| 都市に支店を設立 | 都市に出店・拠点展開 |
| 支店レベル（ATM出張所〜統括本部） | 拠点レベル（個人商店→店舗→主力店舗→地域支社→本社） |
| 他人の支店で利用料 | 他社の拠点で取引（仕入れ・利用代金） |
| インターバンク融資 | **企業間の取引信用（売掛金 / 買掛金）** |
| 自己資本比率・信用格付 | そのまま（中小企業の財務指標として自然） |
| デフォルト連鎖 | 取引先の倒産 → 売掛金の焦げ付き → 連鎖倒産 |
| 金融都市 | 商業都市 |

### 14-2. 取引信用モデル（実装）

`store/gameStore.ts`：

- 取引信用 `credits`（`TradeCredit { supplierId, buyerId, amount }`＝売り手→買い手のエッジ）
- 他社拠点での取引時、現金で払える分は即支払い、**足りない分は買掛金（後払い）**になる（`settleTransaction`）。売り手は同額の売掛金を持つ
- 買掛金には毎ターン金利相当のコスト（6%）がかかり、払いきれないと **資金繰り倒産**
- **連鎖倒産**：倒産した取引先に絡む取引信用は消滅 → 売掛金を持っていた会社は債権（資産）を失い、純資産が割れれば次も倒産（`resolveDefaults`）
- ライバルが全社倒産すれば最後の1社が勝利。総資産は純資産（現金＋拠点＋売掛金−買掛金）で評価
- セレクタ：`payableOf`（買掛金）/ `capital`（自己資本比率・格付）と HUD 表示

> v0.1 の 5 章（銀行・金融システム）・8 章（合併・提携）の銀行固有の記述は、
> この社長コンセプトでは取引信用・業務提携として読み替える（将来更新）。

## 15. 経営投資レイヤー：補助金 / ビジネスマッチングSaaS / DX化（v0.4）

> **狙い：** 現金は基本ためるだけだった状態に「会社そのものへの先行投資」を加え、
> 毎ターン **成長投資 vs 資金繰り** の悩みを作る。連鎖倒産（資金繰り）の緊張に直結する。
>
> **参考座組み：** 地域金融機関が提供する経営支援SaaS（Big Advance／ココペリ）。
> 「ビジネスマッチング・DX支援・補助金情報」を1つのプラットフォームが束ねる構図を、
> SaaS が DX と補助金を後押しする形でゲーム化した。

### 15-1. 3つの要素（`store/gameStore.ts`）

| 要素 | 仕組み | 数値（初期バランス） |
|------|--------|------|
| **DX化** | 投資で全拠点の売上が永続UP（Lv0→3）。`investDX` | +12%/段。費用 200/300/400万 |
| **経営支援SaaS** | 月額サブスクで加入/解約。`toggleSaas` | 月額20万。加入中：取引額+20%・買掛金利息−50%・補助金還元率UP |
| **補助金** | DX投資・過疎地域出店で投資額の一部が即時還元 | 還元率30%（SaaS加入中40%） |

### 15-2. 相互作用と緊張

- 補助金が **DX投資・地方出店のコストを下げる** → DXが売上を底上げ。
- SaaS加入で **取引収入↑・買掛金利息↓・補助金↑** だが、**月額固定費**が資金繰りを締める。
- 先行投資（DX）とSaaS固定費は現金を減らすため、買掛金の利息と合わせて **攻めすぎると資金繰り倒産・連鎖倒産** に呑まれる。
- CPU も拠点規模に応じて DX投資・SaaS加入を行う（`useCpuController`）。

> **将来候補：** SaaS機能の分割課金（マッチング/DX/補助金を個別に）、補助金の採択率（運要素）、
> 業界ストレステストをクライマックスにした勝利条件、提携（複数社の共同DX）。
